Design a high-fidelity enterprise product interface for AGRO-AI.

This is NOT a marketing site.
This is NOT a landing page.
This is NOT a generic SaaS template.
This is NOT a playful dashboard.
This is a serious, premium, minimalist, highly functional enterprise software workspace.

Primary goal:
Design a product interface that feels like a real operational tool used daily by serious enterprise customers. The interface should prioritize work, clarity, decision-making, evidence review, and operational control over marketing language.

Product context:
AGRO-AI is an AI-native enterprise operating system for agriculture. It helps farms, water districts, farmland owners, enterprise agriculture operators, lenders, and corporate teams manage operations, water decisions, evidence workflows, assurance readiness, and agent-assisted work.

The product combines:
- operational monitoring
- data ingestion
- proof / evidence workflows
- assurance / audit-readiness workflows
- AI agent action queues
- reviewer/human approval steps
- exports and reporting

Core product philosophy:
The interface should feel like a real work surface, not a brochure.
Think of the emotional quality of using serious software like Claude, ChatGPT, Linear, Stripe Dashboard, Vercel, Ramp, or modern infrastructure software:
- calm
- fast
- high-signal
- low-clutter
- trusted
- premium
- focused on work
- visually minimal but highly intentional

Main screen structure:

Section 1: Main hero / workspace summary
This is not a marketing hero.
It is a product summary card.

Eyebrow:
AGRO-AI ENTERPRISE PORTAL

Title:
One operating layer for farm evidence, water decisions, assurance, and agents.

Support text:
A focused workspace for proof coverage, irrigation decisions, operational gaps, review gates, and export preparation.

Primary actions:
- Run AGRO-AI Agent
- Attach evidence
- Prepare export

To the right of this summary area, create a compact, premium AI insight panel.
This should feel like the system’s current reasoning state, similar in seriousness to modern AI interfaces.

AI panel title:
What AGRO-AI sees

Primary statement:
Readiness is blocked by missing proof.

Support statement:
The package can be prepared, but external use remains blocked until required records are attached and reviewer checks are completed.

Inside this panel, show concise structured system observations:
- Proof present: Controller events, weather context, crop profile
- Proof missing: Water proof, input applications, traceability
- Next best action: Attach scoped water measurement proof
- Human review: Required before external use

This panel should be compact, high-signal, and clean.

Create four compact metric cards.

Cards:

Each card should have:

Functional design principles:
Everything on screen should support work.
Every module should answer one of these:
- What is happening?
- What matters now?
- What is missing?
- What is the AI doing?
- What should the human do next?
- What is safe to export or share?

The interface should feel like it is built for:
- enterprise operations teams
- growers
- irrigation managers
- audit/review teams
- water districts
- farmland ownership and oversight teams
- finance / reporting stakeholders

The experience should feel:
- premium enough to justify enterprise pricing
- serious enough to show to real buyers
- minimal enough to avoid noise
- clear enough that a user understands the system in under 10 seconds

Do not let visual design overpower function.
Function and clarity come first.
Design quality still matters a lot, but it must serve usability.

Critical quality bar:
This should look like a product a serious enterprise buyer would trust with operational workflows.
The result should look much closer to a polished AI work platform than to a startup homepage.

After generating the Overview screen, keep it editable and structured so it can later expand into:
- Assurance
- Evidence
- Reports
- Agents
using the same design system and layout language.
