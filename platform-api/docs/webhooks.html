Webhooks

Subscribe to events so your systems react the moment intelligence updates — no polling.

Subscribe to events

Register an HTTPS endpoint and the events you care about:

bash
curl -X POST https://api.agroai-pilot.com/v1/webhooks \
  -H "Authorization: Bearer $AGROAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/agroai/webhooks",
    "events": ["irrigation.recommendation.created", "field.updated"]
  }'

The response includes a signing_secret (shown once) — store it to verify deliveries.

Event types

EventFires when
field.updatedA field's attributes or boundary change.
evapotranspiration.updatedNew daily ET is available for a field.
irrigation.recommendation.createdA fresh recommendation is generated.

Delivery payload

POST · application/json
{
  "id": "evt_5Rk0pLm3",
  "type": "irrigation.recommendation.created",
  "created": 1752710400,
  "data": {
    "field_id": "fld_3nZq7Kp2",
    "recommended_depth_mm": 24.0
  }
}

Verify signatures

Every delivery includes an AGROAI-Signature header: a timestamp and an HMAC-SHA256 of timestamp.body keyed with your signing secret. Verify it before trusting the payload.

python
import hmac, hashlib

def verify(secret, header, body):
    parts = dict(p.split("=", 1) for p in header.split(","))
    signed = f"{parts['t']}.{body}".encode()
    expected = hmac.new(secret.encode(), signed, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, parts["v1"])

Reject deliveries whose timestamp is more than five minutes old to defend against replay. Always compare signatures with a constant-time function.

Retries & idempotency

Return 2xx within 10 seconds to acknowledge. Non-2xx or timeouts are retried with exponential backoff for up to 24 hours. Deliveries may repeat, so make handlers idempotent by de-duplicating on the event id.

Idempotency keys (writes)

Send an Idempotency-Key header on POSTs. Retrying with the same key returns the original result instead of creating a duplicate.

bash
curl -X POST https://api.agroai-pilot.com/v1/fields \
  -H "Authorization: Bearer $AGROAI_API_KEY" \
  -H "Idempotency-Key: 5f3a9c1e-8b2d-4c7a-9f10-6d2e1b4a7c88" \
  -H "Content-Type: application/json" \
  -d '{ "name": "North Quarter", "crop": "almond", "centroid": { "lat": 36.33, "lon": -119.29 } }'
