Platform API / Docs / Authentication

Authentication

The Platform API authenticates requests with bearer API keys. Keep them server-side.

API keys

Manage keys in the developer portal under Settings → API keys. Keys come in two flavours:

PrefixEnvironmentData
sk_test_SandboxRealistic sample data, no billing.
sk_live_ProductionYour live workspace data.

Authenticating a request

Send the key as a bearer token in the Authorization header:

bash
curl https://api.agroai-pilot.com/v1/fields \
  -H "Authorization: Bearer $AGROAI_API_KEY"

Requests without a valid key return 401 Unauthorized. A key that is valid but lacks access to a resource returns 403 Forbidden. See Errors.

Keep keys secret

Never expose a key in client-side code. Browser bundles, mobile apps, and public repositories are all readable by anyone. A leaked key can read and write your workspace.

  • Call the API only from your backend; proxy browser and mobile clients through it.
  • Load keys from environment variables or a secrets manager — never commit them.
  • Use sk_test_ keys in development and CI.
  • Scope keys to the minimum access a service needs.

Rotating & revoking keys

Create a new key, deploy it, then revoke the old one — this rotates with zero downtime. Revoke immediately from the portal if a key is ever exposed; revocation takes effect within seconds.

The API Explorer on this site keeps any key you enter in your browser only, sends it directly to the API you choose, and never transmits it to AGRO-AI's web servers. Prefer a sandbox key there.
