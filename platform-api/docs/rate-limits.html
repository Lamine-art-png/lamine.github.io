Rate limits

Limits protect the platform and keep latency low for everyone. Design clients to respect them.

Default limits

PlanRequests / minuteBurst
Sandbox60120
Growth6001,200
EnterpriseCustomCustom

Limits apply per API key. Need more headroom? Talk to us.

Response headers

Every response carries your current budget so you can throttle proactively:

http
RateLimit-Limit: 600
RateLimit-Remaining: 594
RateLimit-Reset: 41

RateLimit-Reset is the number of seconds until the window refreshes.

When you exceed a limit

You receive 429 Too Many Requests with a Retry-After header (seconds). Back off, then retry:

python
import time, requests

def call_with_backoff(fn, attempts=5):
    for i in range(attempts):
        r = fn()
        if r.status_code != 429:
            return r
        wait = int(r.headers.get("Retry-After", 2 ** i))
        time.sleep(wait)
    return r

Add jitter to backoff so retrying clients don't synchronise. Cache reads that don't change minute-to-minute — ET and water balance update daily.
