Pagination

List endpoints return a stable, cursor-paginated envelope. Page forward with the id of the last item you saw.

The list envelope

Lists share one shape: an object of "list", a data array, and a has_more boolean.

200 · application/json
{
  "object": "list",
  "has_more": true,
  "data": [
    { "id": "fld_3nZq7Kp2", "object": "field", "name": "North Quarter — Almonds" }
  ]
}

Parameters

ParameterDescription
limitPage size, 1–100. Defaults to 20.
starting_afterAn object id. Returns the page immediately after it.

Walking every page

python
def all_fields(session, base):
    cursor, out = None, []
    while True:
        params = {"limit": 100}
        if cursor:
            params["starting_after"] = cursor
        page = session.get(f"{base}/fields", params=params).json()
        out.extend(page["data"])
        if not page["has_more"]:
            return out
        cursor = page["data"][-1]["id"]

Cursor pagination is stable under inserts — new items never shift a page you've already read, so you won't see duplicates or skips.
